# 中期展示及文稿

## 展示要求

### 课题简介

1. 项目背景：要求图文结合，讲述课题的由来、背景、意义和挑战等
    
    根据上一次老师以及学长的建议，介绍一下课程设计选题背景以及意义。你是做行程推荐的，那就讲一下做行程推荐可以服务于什么应用。做公共交通服务评价，可以提高传统公交线路规划的效率，使得不会出现某条公交线路过于拥挤/过于清冷，以及公交车在行驶的过程中不会过于拥挤。
    
2. 项目目标：重要，需要将项目抽象成数学问题后的形式化定义
    
    介绍完选题背景、意义之后，介绍选题的问题定义。譬如多人协同游玩地点选址、行程推荐，相关定义就包括，什么是游玩地点，用户怎么定义，推荐会涉及到对游玩地点的打分，那就需要对评分函数做定义，评分函数根据哪些因素确定，为什么这么确定。
    
3. 解决方案：简述项目的解决方案
    
    简介项目的解决思路，**与开题答辩时的思路进行对比**，根据开题答辩老师、学长提的建议，阐述现在定下来的解决方案，并跟开题答辩进行对比，如果解决方案发生了变化，要阐述为什么；
    
    基于什么观测或者先验知识，得到的这个解决方案。判断当前道路状况，如果有单行道，禁停路段、限行路段，就根据步行距离与其他候选上车点到目的地的距离，推荐新的出发点
    

### 开展情况

1. 项目进展：按**时间顺序**或**内容板块**说明课程设计的开展情况；陈述从开题至今做了什么，为什么这么做。
    
    以出租车上车点推荐为例，需要从轨迹数据中提取候选上车点，因此整体的框架就可以分为三个阶段，第一个阶段是数据预处理，第二个阶段要提取出租车状态由空乘转换到载客时的轨迹点，第三个阶段就是对这些轨迹点进行聚类得到上车点的位置，那项目进展就可以从这三个阶段来写，比如已经完成了哪些预处理步骤，以及实现了哪些聚类算法
    
2. 人员分工：所完成工作中人员的分工情况
3. 阶段成果
    - 数据分析：数据预处理及分析，需给出数据分析的结果及结论
        
        数据分析主要是帮助大家理解场景，即拥有更充沛的先验知识。
        在交通流量预测中，流量是因变量，大家就可以尝试做一下流量与各种自变量之间的相关性分析，比如道路属性，道路宽度等等，这就可以帮助大家构建特征。
        
        以及在城市公共交通服务评价中，通过对巴士线路流量的分析，我们可以得出哪些巴士乘坐的人非常多，那我们就可以分流，哪些巴士乘坐的人非常少，那是否就可以暂停运营
        
        如果让大家去做需求预测的数据分析，大家会怎么做。
        
        主要要讲好，数据分析的结果说明了什么，比如你所要达到的目标，跟你分析的数据有什么关系。
        
    - 初步实验结果：如已经采用算法模型进行尝试，可给出算法量化评估结果、可视化评估结果与分析
        
        算法的量化评估结果，可视化分析，以及badcase分析，引申出改进方向
        
4. 未来规划
    
    阐述项目未完成部分的安排、现有解决方案的改进思路等。
